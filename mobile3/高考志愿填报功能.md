# 高考志愿填报系统 MVP 详细功能规格书

## 一、 项目背景与核心逻辑

本系统旨在通过用户录入的高考成绩与意愿，利用后台已有的冲稳保算法，快速生成、筛选并导出志愿方案。

* **核心链路**：录入成绩 → 生成志愿池（3:4:3 分布） → 筛选标注 → 志愿 PK → 草稿管理 → 导出分享。
* **设计原则**：大气、简约、决策导向，采用厚浪动态波浪 UI 风格。

---

## 二、 详细功能模块拆解

### 1. 考生画像录入（核心模块）

| 功能点 | 规则与交互细节 | 备注 |
| --- | --- | --- |
| **1.1 基础信息填报** | 必填：省份、选科（物/史）、总分、全省位次。选填：意向专业。 | 界面采用简约大气的输入控件。 |
| **1.2 校验机制** | 位次必须 > 0 且低于本省总考生数，分数需符合该省满分限制。 | 简化校验逻辑，不阻塞流程。 |
| **1.3 品牌化入口** | 页面顶部常驻直播入口图标；右下角悬浮 1v1 咨询入口。 | 呼应厚浪科技品牌调性。 |

### 2. 志愿匹配结果池（深度优化）

| 功能点 | 规则与交互细节 | 备注 |
| --- | --- | --- |
| **2.1 智能池生成 ★** | 用户输入生成数量 $N$（$N \ge$ 本省可填数）。系统按 **3:4:3** 比例自动从后台抓取“冲、稳、保”院校。 | 解决用户不知道“生成多少才科学”的痛点。 |
| **2.2 三维优先级排序** | 提供“院校优先”、“专业优先”、“职业优先”三个胶囊按钮，点击即刻重排列表。 | 砍掉复杂的细分筛选，聚焦核心意向。 |
| **2.3 院校卡片展示** | 包含：Logo、院校名称、城市（新增）、标签（985/211等）、2024 录取位次、冲稳保标签。 | 增加城市字段，辅助地域决策。 |
| **2.4 高频标注系统 ★** | 每个卡片下方设有：高、中、低、弃选。顶部设“按标注排序”开关，开启后“高/中”意向自动置顶。 | 标注状态在全系统（详情页、草稿页）实时同步。 |

### 3. 志愿 PK 与传播矩阵

| 功能点 | 规则与交互细节 | 备注 |
| --- | --- | --- |
| **3.1 多选对比** | 支持勾选任意数量院校进入 PK 池。对比维度包括：概率等级、位次、省份、办学性质、选科要求。 | 解决院校“二选一”纠结难题。 |
| **3.2 PDF 导出裂变 ★** | 生成 PDF 对比表。**新增规则**：PDF 底部自动附带小程序码及厚浪品牌 Logo。 | 利用 PDF 在家长群的流传实现二次获客。 |

### 4. 志愿草稿管理（排序确认）

| 功能点 | 规则与交互细节 | 备注 |
| --- | --- | --- |
| **4.1 数量配比校验** | 用户设置“冲、稳、保”具体个数。校验规则：冲+稳+保 $\le$ 本省可填总数。 | 引导用户建立科学的填报梯度。 |
| **4.2 拖拽排序 ★** | 草稿列表支持长按手势上下拖拽，实时调整填报顺序。 | 模拟真实填报系统的排序操作。 |
| **4.3 导出/导入** | 支持导出标准格式的 `.xlsx` 表格。导入功能仅识别院校 ID，用于快速同步。 | 方便用户在 PC 端正式填报时对照。 |

### 5. 直播与导师咨询（商业入口）

| 功能点 | 规则与交互细节 | 备注 |
| --- | --- | --- |
| **5.1 1v1 极简咨询** | 文字聊天模式，点击卡片咨询时自动发送该院校信息给导师。 | 后期可扩展语音和预约功能。 |
| **5.2 极简直播** | 仅展示“正在直播”列表。直播间仅保留画面与评论区，砍掉打赏等复杂功能。 | 满足“有问题能随时看”的即时性需求。 |

## 三、 技术与架构建议（后期扩展性）

1. **数据层架构**：标注信息（高/中/低）建议采用独立的 Label 表存储，以便后期增加更多自定义标签（如“离家近”、“王牌专业”）。
2. **API 标准化**：后端返回的“冲稳保”数据需包含 `probability_score`，方便前端后期实现更细腻的概率百分比进度条展示。
3. **UI 统一化**：所有的按钮、卡片、背景动效需符合“厚浪”品牌指南，保持极简大气。
