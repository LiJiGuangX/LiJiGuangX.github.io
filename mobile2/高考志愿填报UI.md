既然我们已经确定了 MVP 的功能逻辑，接下来的 UI 设计需要遵循**“信息高密度、视觉极简、决策导向”**的原则。为了让你和前端开发能快速上手，我为你梳理了“志愿匹配结果页”和“志愿草稿页”这两个核心页面的 UI 字段布局建议。

## 一、 “志愿匹配结果页” UI 布局建议

这个页面是用户停留最久的地方，必须在手机首屏展示出最重要的决策维度。

### 1. 顶部：参数汇总与快速筛选

* **信息条：** 简要显示“山东/物理/620分/15000位”。
* **操作区：** * **志愿池数量：** 一个数字输入框（如：“生成 150 个”）。
* **排序切换：** 院校优先 / 专业优先 / 职业优先（胶囊按钮切换）。
* **一键排序：** “按标注排序” 悬浮按钮。



### 2. 院校志愿卡片（核心组件）

卡片建议采用“左-中-右”或“上下两段式”结构：

* **左侧/顶部：** 院校 Logo + 院校名称（如：武汉大学）。
* **中部核心字段：**
* **标签组：** 985/211、双一流、公办、湖北·武汉。
* **专业信息：** 展示用户意向专业（或该校王牌专业）及选科要求。


* **右侧/底部决策区：**
* **概率标签：** **“冲”**（红色）、**“稳”**（黄色）、**“保”**（绿色）。
* **位次对比：** “24年位次：12500”（直观对比用户当前的 15000 位）。


* **底部操作栏：**
* **标注选择：** 高 / 中 / 低 / 弃（四个圆圈点选）。
* **功能入口：** 详情、咨询导师、加入对比。

## 二、 “志愿草稿/导出页” UI 布局建议

该页面的核心目标是“确认顺序”和“分享决策”。

### 1. 列表展示

* **序号：** 1、2、3...（对应填报系统序号）。
* **拖拽区：** 左侧显示“≡”图标，支持长按上下拖动调整顺序。
* **关键冲突提醒：** 若总数超过本省限制，顶部显示红色 Tip：“当前已选 105 个，超出本省 96 个限制”。

### 2. 底部动作条 (Sticky Bottom)

* **左侧：** “PK对比 (已选N)”
* **右侧：** “导出 PDF” 或 “导出 Excel”。

## 三、 UI 视觉风格建议（针对“厚浪”品牌）

根据你之前的偏好，我们采用以下设计语言：

* **背景色：** 纯白或浅灰底色，保持氛围“大气、简洁”。
* **品牌色：** 建议使用**“深邃蓝”**作为主基调，搭配动态波浪背景（Wave Background）作为页面顶部点缀，呼应“厚浪（Houlang）”品牌。
* **间距：** 增加卡片间的呼吸感，避免高考报名的紧张压抑感。
* **动效：** 在标注“高/中/低”或调整顺序时，加入轻微的触感反馈（Haptic Feedback）和淡入淡出。

---

## 四、 给后端/前端的开发 Tip

1. **PDF 导出：** 建议前端使用 `html2canvas` 或后端调用极简的 PDF 模板引擎，确保导出的表格在 A4 纸比例下清晰。
2. **本地存储：** 用户的标注状态（高/中/低）应实时同步到本地 `Storage`，防止误触返回导致进度丢失。
