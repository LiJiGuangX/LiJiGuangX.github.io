1. 产品概述


1.1 背景



高考志愿填报是影响学生未来发展的重要决策，但大多数家庭缺乏专业指导。本平台旨在连接具备相关院校/专业经历的高校在读生与应届高考学生，通过结构化的咨询服务帮助学生科学规划志愿。



1.2 产品定位



面向 B2C 的移动端知识服务平台，核心模式为：

学生付费预约 → 导师在线一对一咨询 → 平台抽佣（15%）



1.3 核心价值主张



| 角色 | 核心价值 |

|------|----------|

| 学生 | 获取真实经验者的个性化建议，科学填报志愿 |

| 导师 | 利用自身学习经历变现，积累指导经验 |

| 平台 | 撮合交易，维护服务质量，建立可信生态 |



1.4 入口结构



index.html 作为主入口，提供三个子系统的跳转：



志愿填报工具 → gaokao.html（学生使用）
导师端 → tutor.html（咨询师使用）
管理后台 → admin.html（平台运营使用）



> 学生端 (student.html) 入口已预留，尚未实现。

2. 系统架构与用户角色


2.1 技术架构


前端技术：纯 HTML5 + CSS3 + 原生 JavaScript，无框架依赖
设计基础：移动端优先，最大宽度 390px，模拟微信/支付宝小程序效果
共享资源：styles.css（全局设计系统）、config.js（全局配置常量）
路由机制：单页面内多 .page 切换（display:none / display:block），配合底部导航栏
数据存储：当前为前端内存变量（const / let），无后端持久化


2.2 用户角色



| 角色 | 入口 | 权限范围 |

|------|------|----------|

| 高考学生 | gaokao.html | 录入信息、筛选院校、规划志愿、预约导师 |

| 咨询导师 | tutor.html | 管理预约、设置时间、维护个人信息、查看收益 |

| 平台管理员 | admin.html | 全量数据查看、审核内容、处理争议、管理导师账号 |

3. 公共 UI 规范


3.1 顶部导航栏 (Topbar)


固定在页面顶部，z-index: 100
蓝色主题（#1e40af）用于导师端和志愿工具
橙色主题（#d97706）用于学生端
左侧返回按钮（←）或跳转上级页面
中间显示当前页面标题（随底部导航切换动态更新）


3.2 底部导航栏 (Bottom Nav)


固定在底面，5个标签项
激活状态：文字变蓝（#1e40af）、图标高亮
切换时同步更新顶部标题和内容区


3.3 弹窗 (Modal/Overlay)


半透明遮罩层（rgba(0,0,0,.5)），从底部滑出
圆角容器（border-radius: 20px 20px 0 0）
最大高度 72vh，内容可滚动
点击遮罩区域关闭弹窗


3.4 Toast 提示


出现在底部导航栏上方，黑色半透明圆角背景
自动消失（2.4 秒）
用于操作反馈（确认、警告、错误）


3.5 徽章体系 (Badge)



| 样式类 | 颜色 | 使用场景 |

|--------|------|----------|

| bdg-gold | 黄底橙字 | 金牌/资深导师等级 |

| bdg-green | 绿底绿字 | 已完成、审核通过 |

| bdg-gray | 灰底灰字 | 初/中级导师、已完成 |

| bdg-red | 红底红字 | 违规、封号状态 |



3.6 设计语言


主色：#1e40af（深蓝）

5.2 页面①：预约管理


5.2.1 搜索与过滤



| 控件 | 功能 |

|------|------|

| 文字搜索框 | 按学生姓名或咨询问题关键词实时过滤 |

| 日期选择器 | 选择具体日期后只显示该日的预约 |

| 状态筛选 Tab | 全部 / 待确认 / 已确认 / 已完成 / 已取消 |




三个过滤条件同时生效，互相叠加。



5.2.2 待确认视图（档期分组）



「待确认」状态的预约按档期（日期 + 时间段）进行分组显示：




档期组头：

📅 日期 + 时间段（如 2026-06-10 19:00-19:30）
橙色背景，右侧显示「N 人待确认」徽章



预约卡片（组内每人一张）：

学生姓名（脱敏，如 李**）
实时倒计时：显示距离系统规定响应截止时间（10分钟响应要求）的剩余时长，格式 ⏱ 剩 M:SS
剩余 > 5 分钟：灰色
剩余 2–5 分钟：橙色警告
剩余 < 2 分钟：红色紧急
已超时：显示 ⚠️ 超时待处理
倒计时每秒刷新（setInterval）
咨询问题内容（卡片式展示）
接受预约 按钮（蓝色）→ 状态变为「已确认」，Toast 通知
取消预约 按钮（灰色）→ 触发取消原因弹窗


5.2.3 其他状态视图（平铺列表）



「已确认 / 已完成 / 已取消」展示为平铺卡片：



学生姓名 + 状态徽章
日期时间段
咨询问题摘要
已确认：「发起会议」+ 「取消预约」按钮
已完成：「查看笔记」按钮（待实现）
已取消：无操作按钮


5.2.4 取消预约弹窗



导师取消预约时必须选择原因（单选），包含：




| 选项 | 取消原因 |

|------|----------|

| 1 | 时间冲突，档期临时有变 |

| 2 | 该话题超出我的专业范围 |

| 3 | 预约信息不完整，无法充分准备 |

| 4 | 个人紧急情况 |

| 5 | 其他原因 |



补充说明（文本框，选填）
必须选择原因才能点「确认取消」，否则 Toast 提示
确认后：该预约状态变为「已取消」，学生收到通知（待实现），列表刷新

5.3 页面②：时间管理


5.3.1 费率说明卡片（只读）



展示当前等级对应的系统定价，导师不可修改：




| 时长 | 费率（银牌等级示例）| 解锁条件 |

|------|---------------------|----------|

| 15 分钟 | ¥ 30 | 所有等级可用 |

| 30 分钟 | ¥ 80 | 所有等级可用 |

| 60 分钟 | ¥ 130 | 所有等级可用 |

| 90 分钟 | ¥ 180 | 金牌及以上等级解锁 |




不同等级对应不同费率档位（系统自动设定）。



5.3.2 交互式日历


支持月份前进/后退导航
有时段的日期：显示蓝色背景（.av 样式），表示该日有可预约时段
日历渲染时自动计算：
若某时间段设置了「循环日期」，则该月所有匹配星期的日期格标蓝
若某时间段为「仅此日」，则仅对应日期标蓝
当前选中日期：深蓝背景（.sel 样式）
点击日期：
更新「当日时间段」卡片标题（日期 + 星期）
显示「添加」按钮
渲染该日对应的所有时间段（循环 + 一次性）
图例说明：蓝色实心圆 = 有可预约时段 / 黄底圆 = 当前选中


5.3.3 当日时间段管理



时间段列表（按开始时间升序排列）：

每行：起始时间 — 结束时间 + 循环标签（如 ↻ 工作日）+ 月份标签（如 3月 4月）
副文字：「循环时间段」或「仅此日」+ 时长（分钟）
右侧 ✕ 按钮删除该时段（同步更新日历和时间图谱）



添加时间段表单（点击「添加」展开，隐藏「添加」按钮）：




| 字段 | 控件 |

|------|------|

| 开始时间 | 小时选择（00–23）+ 分钟选择（00/15/30/45） |

| 指导时长 | Chip 单选：15分钟 / 30分钟 / 45分钟 / 60分钟 / 90分钟 |

| 重复日期 | 7个圆形圈按钮（一~日），可多选 |

| 快捷预设 | 工作日 / 每天 / 清除 |

| 应用月份 | 月份 Chip（3–8月），可多选，不选默认仅本月 |




校验规则：

未选日期时点添加 → Toast 提示选择日期
同一日期同一时间点已存在 → Toast 提示冲突



保存逻辑：

有选循环日期 → 生成循环时间段（weekdays 数组非空，date 为 null）
无循环日期 → 生成一次性时间段（weekdays 为空，date = 当前选中日期）
保存后：隐藏表单，刷新时间段列表，刷新日历标记，刷新时间图谱


5.3.4 本周时间分布图



一张可视化甘特图，展示循环时间段在一周各天的分布：



纵轴：周一至周日（7行）
横轴：6:00 – 22:00（16小时范围），每 2 小时一个刻度标注
红色填充块：该天该时间段内有可预约时段（宽度按时长百分比计算）
仅循环时间段（有 weekdays 设置）显示在图谱中，一次性时间段不计入
无循环时间段时显示引导文字
添加/删除时间段后实时刷新

8. 核心业务规则


8.1 预约流程


PLAINTEXT
学生选择导师 → 选择档期 → 填写咨询问题 → 支付费用
    → 预约进入「待确认」队列
    → 导师 10 分钟内必须响应（接受或取消）
    → 超时未响应：系统自动取消，全额退款，记入响应率指标
    → 导师接受：状态变「已确认」，双方收到确认通知
    → 到时发起会议（视频/电话）→ 完成后状态变「已完成」
    → 导师收入到账（扣除 15% 平台服务费）


8.2 费率规则


费率由平台按导师等级统一设定，导师不可自定义
当前已定义 15 / 30 / 60 / 90 分钟四档
90 分钟档仅金牌（Lv5）及以上等级可开启
不同等级对应不同单位费率（等级越高费率越高，具体数值待定）


8.3 取消与退款规则



| 取消方 | 取消时间 | 退款比例 |

|--------|----------|----------|

| 学生 | 距预约 > 24 小时 | 全额退款 |

| 学生 | 距预约 ≤ 24 小时 | 部分退款（待定） |

| 导师 | 任意时间 | 学生全额退款，导师记入违规 |

| 系统（超时） | 导师 10 分钟未响应 | 学生全额退款，导师响应率扣分 |



8.4 等级晋升规则


升级需同时满足当级所有指标要求
所有指标取近 3 个月滚动数据计算
有效违规记录（近 3 个月）会阻断升级申请
系统定期（月底）自动重新评估所有导师等级
指标持续未达标可触发自动降级（具体降级触发条件待定）


8.5 荣誉认证规则



| 认证类型 | 触发方式 | 审核时效 | 到期/撤销 |

|----------|----------|----------|-----------|

| 985/211 认证 | 注册时自动触发 | 随资质审核同步 | 永久有效 |

| 优质咨询师 | 系统自动检测达标后触发 | 1-2 工作日 | 指标下降后可撤销 |

| 年度 TOP | 导师主动申请 | 5 工作日 | 年度有效 |

| 职业资格 | 导师上传证书 | 3 工作日 | 到证书有效期 |

| 命中率认证 | 导师上传案例 | 人工审核 | 长期有效 |



8.6 内容审核规则



所有对外展示内容（简介 / 关键词 / 宣传素材）均需经过审核后方可上线：



导师提交 → 进入平台审核队列
AI 预审：自动标记风险词汇、评估风险等级
人工复审：审核员做最终判定（通过 / 要求修改 / 拒绝）
审核结果通知导师（站内信或短信）
已通过的内容若被举报，进入复审流程


8.7 志愿池匹配规则



院校冲/稳/保分类依据：




| 分类 | 判断条件 |

|------|----------|

| 冲一冲 | 学生分数 < 院校2024录取分，概率 < 50% |

| 稳一稳 | 学生分数 ≈ 院校2024录取分，概率 50–80% |

| 保一保 | 学生分数 > 院校2024录取分，概率 > 80% |




专业匹配过滤：

若学生已选意向专业 → 仅显示包含匹配专业的院校
若未选意向专业 → 显示全部院校，专业数量统计基于全量专业
