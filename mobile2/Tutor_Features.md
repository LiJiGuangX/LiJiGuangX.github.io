## 1. 导师注册与认证体系

### 1.1 账号注册阶段

**目的：** 建立平台用户账户，为导师身份认证做准备。

**流程：**

1. 用户访问平台，选择"导师注册"
2. 输入手机号/邮箱和密码，验证（短信验证码）
3. 注册完成，自动进入"资质认证"流程

**系统状态：** 账号注册成功后，导师进入 `status: registered` 状态，部分功能自动禁用（见1.3权限表）。

---

### 1.2 资质认证阶段（初始等级分配）

**页面：咨询师注册认证页**

**目的：** 验证导师身份真实性和学历背景。这是入场的必要条件，直接影响能否对用户展示、能否接受预约。学历验证仅用于身份确认，不决定初始等级。

**采集信息（全部必填）：**

| 字段          | 规则                 | 说明                                                                    |
| ------------- | -------------------- | ----------------------------------------------------------------------- |
| 姓名          | 文本，≤30字         | 需与身份证一致；用于显示在导师卡片上                                    |
| 身份证号      | 18位数字             | 系统验证格式；用于学信网核实学历                                        |
| 毕业/在读院校 | 选择式 + 自由输入    | 调用院校库（[补充]完整院校库）；不在库内可自由输入，但需运营手工审核    |
| 所学专业      | 选择式 + 自由输入    | 基于选中院校的专业列表；自由输入需审核                                  |
| 学历起止时间  | 日期选择             | [补充]是否需要？用于验证在读/毕业状态                                   |
| 学信网验证    | 上传截图或输入验证码 | 系统自动验证；[补充]具体验证API集成文档。这里有个在线验证码，可以验证。 |

**认证流程：**

1. 导师填写所有必填信息并上传学信网验证材料
2. 点击"提交认证"
3. **自动验证层**：系统调用学信网API验证身份和学历
   - ✓ 验证通过 → 进入步骤4
   - ✗ 验证失败 → 显示错误提示，允许重新上传材料
4. **进入"认证审查中"状态** (status: authenticating)
   - [补充]审核周期标准？建议1-3个工作日
   - 前端提示："您的认证正在审查中，请等待平台通知"
5. **运营手工审核**：检查是否真实学历、是否符合平台要求
6. **审核结果反馈**：
   - ✓ **通过** → status: authenticated，自动分配初始等级，跳转至"个人宣传设置"
   - ✗ **驳回** → status: rejected + reject_reason（显示原因），允许重新提交

**初始等级分配规则**：

**所有导师认证通过后统一从「初级」开始**，与学历无关。初级等级作为"新人期"，导师需要通过积累成长值和达成KPI指标逐步升级至中级、高级等。

此设计的目的：
- 确保平等的起始点，激励所有导师提升服务质量
- 避免学历歧视或倒挂（如高学历却服务差的现象）
- 让用户看到真实的服务能力而非学历背景

**初始规则表**（以下已过期，需更新）：

| 等级 | 名称 | 咨询价建议 | 曝光权重 |
| ---- | ---- | ---------- | ------- |
| Lv1  | 初级 | 50-100元/30min | 1.0x |

学历信息保存在导师资料中，用于用户信任评估，但不直接影响等级和收费。

**关键决策：**

- [待确认] 初始等级是否锁定不变？还是后续可通过升降机制调整？
- [待确认] 学历验证失败（学信网无记录）的导师如何处理？拒绝？还是允许其他方式验证？

### 1.3 认证状态与权限矩阵

导师在系统中的权限随认证状态而变化。以下矩阵展示了不同状态下各核心功能的可用性：

| 功能/权限              | 注册中 (registered) | 认证审查中 (authenticating) | 已认证 (authenticated) | 认证被拒 (rejected) |
| ---------------------- | ------------------- | --------------------------- | ---------------------- | ------------------- |
| **编辑基础信息** | ✓ 可编辑           | ✓ 可编辑                   | ✓ 可编辑              | ✓ 可编辑           |
| **设置宣传信息** | ❌ 不可见           | ✓ 可编辑，不展示           | ✓ 编辑立即展示        | ✓ 可编辑，不展示   |
| **设置预约时间** | ❌ 禁用             | ❌ 禁用                     | ✓ 可用                | ❌ 禁用             |
| **对用户端展示** | ❌ 隐藏             | ❌ 隐藏                     | ✓ 展示                | ❌ 隐藏             |
| **接收预约请求** | ❌ 禁用             | ❌ 禁用                     | ✓ 可用                | ❌ 禁用             |
| **预约管理页**   | ❌ 隐藏             | ❌ 隐藏                     | ✓ 可用                | ❌ 隐藏             |
| **等级成长页**   | ❌ 隐藏             | ❌ 隐藏                     | ✓ 可用                | ❌ 隐藏             |

**各状态详解：**

**认证审查中（authenticating）：**

- 导师可进入"个人宣传设置"页面，编辑所有信息（自我介绍、咨询范围、关键词、价格等）
- 前端顶部显示黄色警告条：⚠️ "您的认证正在审查中，请等待结果。宣传信息已准备好；审核通过后将自动发布"
- 用户端不展示该导师信息（搜索、列表、详情页均隐藏）
- 不能设置预约时间（因为还无法接受预约）

**已认证（authenticated）：**

- 解锁全部功能
- 宣传信息立即展示在用户端
- 可以设置预约时间段
- 开始计算KPI指标（响应率、完成率、好评率）

**认证被拒（rejected）：**

- 显示驳回原因（供导师改进）
- 允许重新提交认证（[澄清]是完全重新填写还是编辑原有信息？建议：编辑原有信息+重新提交）
- 宣传信息权限同"认证审查中"（可编辑但不展示）
- 不能接受预约

## 2. 个人宣传设置模块

### 2.1 模块概述

**页面名称：** 咨询师信息设置

**目的：** 导师建立个人品牌形象。包含自我介绍、咨询范围、关键词、宣传材料、定价等，这些信息直接影响用户搜索、转化率和咨询价格。

**可用时机：**

- ✓ 认证审查中（信息可编辑但不展示）
- ✓ 已认证（信息编辑即刻展示）
- ✓ 认证被拒（重新编辑后重新提交认证）

**布局：** PC端左右分栏（左表单，右预览），移动端垂直排列

---

### 2.2 关键决策与标注

本模块各字段的审核规则、展示时机、与等级的关系，多处需要明确决策：

| 字段       | 必填 | 审核     | 认证中展示 | 已认证展示 | 与等级关系   |
| ---------- | ---- | -------- | ---------- | ---------- | ------------ |
| 自我介绍   | ✓   | ❌       | ❌         | ✓立即     | 无           |

  - [待确认]是否每次编辑都需重新勾选？建议：首次需勾选；之后可不再勾选（假设已永久同意）

## 5. 预约管理流程

### 5.1 预约状态流转

预约在导师端的完整生命周期：

```
用户提交预约 & 支付
      ↓
[待确认] (10 min timeout)
    ├─ 导师接受 → [已确认]
    ├─ 导师拒绝 → [已取消] (100% 退款)
    └─ 10min超时 → [已取消] (100% 退款)
      ↓
[已确认] (waiting for appointment time)
    ├─ 导师取消(违约) → [已取消] (100% 退款，扣50%手续费，成长值-30)
    ├─ 用户取消 → [已取消] (按规则退款)
    └─ 到时进行咨询 → [in progress]
      ↓
[进行中] (video call)
    ├─ 结束 → [已完成] (成长值+10，自动生成会议笔记)
    └─ 中途取消 → [已取消]
      ↓
[已完成]
    └─ 等待用户反馈
```
